# Cloud Ops Manager v2.0.1 任务清单

> **版本主题：** 能放心装、愿意留 — 分发信任 + 体验抛光 + 稳定性  
> **目标周期：** 1～2 周（单人可完成）  
> **北极星：** 新用户从安装到完成一次运维操作（EC2 启停 / OBS 上传）≤ 10 分钟，无 `xattr` 焦虑  

---

## 里程碑概览

| 优先级 | 主题 | Issue 数 | 预估 |
|--------|------|----------|------|
| P0 | 分发与信任 | 4 | 3～5 天 |
| P1 | 体验抛光 | 4 | 2～3 天 |
| P2 | 工程质量 | 3 | 1～2 天 |

**建议发版条件：** P0 全部完成 + P1 至少 3 项 + CI 全绿。

---

## P0 — 分发与信任（必做）

### #1 补全 GitHub 产品截图

**描述：** `image/` 目录补 6～8 张截图，更新 `cloud-ops-manager` 仓库 README 的模块展示区。

**验收标准：**
- [ ] `overview.png` — 仪表盘 + 厂商切换
- [ ] `ec2-list.png` — AWS EC2 列表
- [ ] `s3-objects.png` — S3 对象浏览
- [ ] `huawei-ecs.png` — 华为 ECS 列表/详情
- [ ] `obs-browser.png` — OBS 对象浏览
- [ ] `settings-credentials.png` — 凭证管理页
- [ ] README.zh-CN / README 中图片可正常显示

**预估：** 0.5 天（含截图 + push 到 release 仓）

---

### #2 macOS 应用公证（Notarization）

**描述：** 解决 Gatekeeper 拦截，Release 说明可删除 `xattr` 步骤。

**验收标准：**
- [ ] `electron-builder.yml` 配置 `notarize`（Apple ID + app-specific password 或 API Key）
- [ ] CI/本地打包产出已公证的 `.dmg`
- [ ] 在全新 macOS 上双击安装无需 `xattr`
- [ ] Release 说明更新 macOS 安装指引

**依赖：** Apple Developer 账号  
**预估：** 1～2 天（含证书与首次公证调试）

---

### #3 Windows 代码签名

**描述：** 消除 SmartScreen「未知发布者」警告。

**验收标准：**
- [ ] `electron-builder.yml` 配置 `sign` / `certificateFile` 或 CI 密钥注入
- [ ] `.exe` 安装包有有效 Authenticode 签名
- [ ] Release 附件替换为签名版

**依赖：** EV 代码签名证书（或临时自签 + 说明）  
**预估：** 1 天

---

### #4 集成 electron-updater 自动更新

**描述：** 客户端检测 GitHub Release 并提示/静默下载更新。

**验收标准：**
- [x] 依赖 `electron-updater`，`electron-builder.yml` 配置 GitHub publish
- [x] 设置页检查/下载/重启安装 + 启动后自动检查
- [x] 打包流程产出 `latest*.yml`（electron-builder 默认）
- [ ] 发版时上传 `latest.yml` / `latest-mac.yml` / `latest-linux.yml` 到 Release（供校验）
- [ ] macOS 应用内更新需在 Release 附加 `*-mac.zip`（dmg 不支持 in-app update）

**涉及：** `src/main/updater.ts`、`Settings/UpdateSettings.tsx`  
**预估：** 1～1.5 天（核心已完成）

## P1 — 体验抛光（高 ROI）

### #5 S3 预签名 URL UI

**描述：** 后端已有 `get-signed-url`，补前端入口。

**验收标准：**
- [x] S3 对象行/详情增加「复制预签名链接」
- [x] 可选有效期（15min / 1h / 24h）
- [x] 复制成功 toast + 剪贴板
- [x] i18n zh-CN / en-US

**涉及文件：** `S3ObjectBrowser.tsx`、`s3.ipc.ts`（已有）、`i18n`  
**预估：** 0.5 天

---

### #6 华为 RDS 规格人类可读展示

**描述：** 与 EC2/ECS 一致，隐藏 `rds.mysql.s1.large` 类原始 flavor。

**验收标准：**
- [x] `RDSPage` / `RDSInstanceDetail` 显示如 `2 vCPU / 4 GB`
- [x] 列表列宽与 ECS 风格一致
- [x] 无 flavor 元数据时显示 `-`

**涉及文件：** `instanceSpec.ts`（扩展）、`RDSPage.tsx`、`RDSInstanceDetail.tsx`  
**预估：** 0.5 天

---

### #7 OBS 目录大小 / 修改时间聚合展示

**描述：** 二级目录 CommonPrefixes 无 API 元数据，通过 prefix 下对象聚合。

**验收标准：**
- [x] `obs.ts` `listObjects` 对文件夹聚合 `size` + `max(lastModified)`
- [x] 前端文件夹行显示聚合值（非 `-`）
- [x] 对象数 >1000 时显示「部分统计」或 loading，不卡死 UI
- [x] README 常见问题更新说明

**预估：** 1 天

---

### #8 Release 仓 README 与 v2.0.1 说明对齐

**描述：** `cloud-ops-manager` 公开仓与每次 Release 同步。

**验收标准：**
- [x] 安装表与 Release 附件文件名一致（见 `RELEASE-v2.0.1.md`）
- [x] 版本号、更新说明、免责声明统一
- [ ] 发版时 push 到 `cloud-ops-manager`（需本机终端提交，避免 Co-authored-by）

**预估：** 0.25 天 / 每次发版
